# Re:Campus — 프로젝트 설명서

## 1. 프로젝트 개요
Re:Campus는 캠퍼스 안에서 버려지는 전공책·의류·생활용품·전자기기를 학생 간 재사용으로 연결하고,
**거래 결과를 탄소 절감량과 폐기물 감소량 데이터로 시각화**하는 **캠퍼스 자원 순환 데이터 플랫폼**이다.

## 2. 창의성 포인트
- **컨셉 전환**: 중고거래 앱이 아닌 **"학교 ESG 데이터 플랫폼"**으로 포지셔닝.
- **거래 = 데이터**: 거래가 완료되는 순간이 곧 환경 데이터 적립 시점. 사용자 행동이 학교 차원의 ESG 지표로 환산된다.
- **임팩트 환산 문구**: 단순한 kg 수치 대신 *"승용차 n km 절감 / 소나무 n그루 흡수량"*으로 체감 가능한 가치를 전달.
- **멀티 캠퍼스 비교 대시보드**: 학교 간 자원 순환 성과를 정량 비교하여 ESG 경쟁을 유도.

## 3. 기술 적용 내용
| 항목 | 적용 |
|---|---|
| Framework | React 18 (함수형, Hooks 기반) |
| Build | Vite 5 (HMR, 빠른 dev) |
| Routing | React Router v6 (`<Routes>`, 동적 파라미터 `/items/:id`, 권한 가드) |
| Styling | Tailwind CSS 3 + 커스텀 brand 그린 팔레트, `@layer components`로 디자인 토큰화 |
| State | React Context (현재 사용자/토스트/새로고침 트리거) + localStorage CRUD |
| Charts | Recharts (Bar, Pie, Line + ResponsiveContainer) |
| 재사용성 | `utils/`(storage·calcEco·format·validation), `hooks/useLocalStorage`, 8개 컴포넌트 |
| 데이터 | `data/seedData.js` 시드 12 아이템 + 6 사용자 + 7 신청 |
| 폼/검증 | 카테고리별 자동 절감량 미리보기, 거래 유형에 따른 가격 분기, 클라이언트 유효성 검증 |

## 4. UI 설계 의도
- **그린 ESG 브랜드 시스템** — `brand-50 ~ brand-900` 단일 팔레트로 일관성 확보
- **카드 중심 레이아웃** — 정보 위계가 명확하고 모바일에서도 자연스럽게 스택됨
- **상태 칩** — 거래 가능/예약중/완료/숨김을 한눈에 색상으로 구분
- **임팩트 배너** — 추상적 숫자를 직관적 비유로 전환
- **빈 상태 / 토스트 / 모달 일관성** — 사용자 피드백 누락 없음
- **모바일 햄버거 메뉴 + 그리드 반응형** — 시연 환경(데스크톱·모바일) 모두 깨지지 않음

## 5. 완성도 설명
- `npm install` → `npm run dev` 즉시 실행
- 첫 실행 시 시드 자동 적재, 새로고침에도 데이터 유지
- **데이터 초기화 버튼** 1클릭으로 시연 직전 상태 복구
- 빈 상태/검증 실패/네트워크 무관 — 콘솔 에러 0건
- 라우트 기반 권한 가드 (`/items/new`, `/my`, `/admin`)
- 본인 물품 자기 신청 방지, 중복 신청 방지, 상태 변경 시 토스트 알림

## 6. 기대 효과
- **학생** — 환경 기여를 정량 체감, 자원 재사용 동기 부여
- **학교** — ESG 활동을 정량적으로 측정·홍보·보고서화
- **사회** — 캠퍼스 단위 순환 경제 모델 확산 (학교 간 비교를 통한 ESG 경쟁)

## 7. 한계점과 향후 개선 방향
| 한계 | 개선 방향 |
|---|---|
| 환경 계수가 데모용 추정값 | 카테고리별 LCA 실측 계수 적용 (산업통상자원부·환경부 데이터) |
| localStorage 단일 디바이스 | Supabase / Firebase 등 BaaS 연동으로 실시간 동기화 |
| 학교 인증 수동 | 학교 메일 도메인 OTP 자동 인증 |
| 알림 기능 부재 | FCM 푸시 + 학교 카카오톡 채널 연동 |
| 분석 단순 | 학기별·전공별 통계, 학생회 ESG 리포트 PDF 자동 생성 |
