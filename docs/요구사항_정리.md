# Re:Campus — 요구사항 정리

## 1. 프로젝트 주제
캠퍼스 자원 순환 데이터 플랫폼

## 2. 세부 주제
캠퍼스에서 버려지는 전공책·의류·생활용품·전자기기를 학생 간 재사용으로 연결하고,
거래 결과를 **탄소 절감량·폐기물 감소량 데이터**로 시각화하는 학교 ESG 플랫폼.

## 3. 문제 정의
- 매 학기 막대한 자원이 폐기되지만 학교/학생 단위 자원 순환 성과는 측정되지 않는다.
- 일반 중고거래 앱은 캠퍼스 ESG 지표로 환산되지 않으며, 학교 차원의 의사결정에 활용되지 못한다.
- 학생들도 "내가 얼마나 환경에 기여했는지"를 체감하지 못한다.

## 4. 주요 사용자
- **재학생 (일반 사용자)** — 물품 등록, 신청, 수령
- **학교 관리자** — 학교 메일 인증 승인, 부적절 게시물 관리
- **학교 ESG 담당 부서** — 자원 순환 데이터 모니터링 (대시보드)

## 5. 핵심 요구사항
1. 거래 결과가 자동으로 환경 데이터(탄소·폐기물)로 환산되어 대시보드에 적립되어야 함
2. 학교별·카테고리별·월별 통계가 시각화되어야 함
3. 학교 메일 기반 인증 상태 관리(관리자 승인)가 가능해야 함
4. 발표 시연에서 어떤 단계에서도 흐름이 끊기지 않을 만큼 안정적이어야 함

## 6. 기능 요구사항
### 사용자
- 데모 로그인 / 사용자 전환
- 홈 (누적 재사용 건수, 탄소 절감, 폐기물 감소, CTA 3종)
- 물품 둘러보기 (검색, 카테고리/상태/거래유형 필터, 카드 리스트)
- 물품 상세 (예상 절감량, 수령 신청, 등록자 본인 시 신청자 관리 / 거래 완료)
- 물품 등록 (이미지 업로드 또는 URL, 카테고리 선택 시 절감량 자동 미리보기)
- 마이페이지 (등록·신청·완료, 내 누적 기여량)

### 관리자
- 사용자 목록 + 학교 인증 승인/대기 토글
- 게시물 목록 + 숨김 처리/복구
- 데이터 초기화

### 대시보드
- 누적 지표 4종 + 임팩트 환산 문구
- 학교별 거래 수 / 탄소 절감 차트
- 카테고리 비중 차트
- 월별 추이 (거래 수 + 탄소 절감)
- 카테고리별 상세 테이블

## 7. 비기능 요구사항
- 새로고침해도 데이터 유지 (localStorage)
- 모바일/태블릿/데스크톱 반응형
- 콘솔 에러 0건
- 첫 실행 시 시드 데이터 자동 적재
- 데이터 초기화 1클릭

## 8. 데이터 구조
### users
| 필드 | 설명 |
|---|---|
| id | 고유 ID |
| name | 이름 |
| email | 학교 메일 |
| school | 소속 학교 |
| role | `user` / `admin` |
| verificationStatus | `approved` / `pending` |

### items
| 필드 | 설명 |
|---|---|
| id, userId, title, description, category, condition | 기본 정보 |
| transactionType | `나눔` / `저가거래` |
| price, imageUrl | 가격, 이미지 |
| status | `available` / `reserved` / `completed` / `hidden` |
| wasteReductionKg, carbonReductionKg | 환경 절감량 |
| createdAt, completedAt | 일시 |

### requests
| 필드 | 설명 |
|---|---|
| id, itemId, requesterId, ownerId | 관계 |
| status | `pending` / `accepted` / `rejected` / `completed` |
| createdAt | 신청일 |
